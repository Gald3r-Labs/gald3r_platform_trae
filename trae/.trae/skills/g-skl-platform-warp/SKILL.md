# g-skl-platform-warp

Activate for: setting up gald3r with Warp terminal, configuring Warp AI workflows, understanding Warp's agent mode, or verifying Warp gald3r integration.

---

## 1. Platform Overview

**Warp** is an AI-native terminal (not an IDE) with built-in AI assistance. Its AI surface (2026)
splits into **Active AI** (always-on inline hints), **Agent Mode** (interactive multi-step terminal
tasks), and **Cloud Agents** (autonomous, event-triggered). It has **no file tree / editor pane** —
its primitives are terminal sessions, project **Rules** files, and **Warp Drive** (cloud store of
Workflows, Notebooks, Prompts, Environment Variables).

- **Active AI / Agent Mode**: natural language → terminal commands and multi-step execution
- **Warp Drive**: shareable Workflows and Notebooks (cloud-backed); agents pull context from it
- **Project Rules**: `AGENTS.md` / `WARP.md` at the repo root **auto-apply** within the project —
  the direct equivalent of Cursor rules / Claude Code's `CLAUDE.md` (corrects the prior claim that
  Warp has "no project-level rules file")
- **MCP**: first-class; auto-discovers existing `~/.claude.json` / `.mcp.json` / `.codex/config.toml`

**gald3r target tier**: AI terminal. Integration via root `AGENTS.md`/`WARP.md` rules + shared MCP
config; Warp Drive Workflows are a manual/cloud-only adjunct.

> **Full capability assessment**: see `PLATFORM_SPEC.md` (9 sections) in this skill folder.

## 2. Config File Layout

Warp **does** have a project-level rules file. The primary on-disk integration point is the
root rules file (`AGENTS.md` / `WARP.md`); the rest of Warp's configurable state is cloud-backed
(Warp Drive) or app-managed.

```
<project-root>/
├── WARP.md             ← project rules (ALL-CAPS); takes priority if both exist — gald3r writes
├── AGENTS.md           ← Warp's default project rules (generated by /init) — gald3r writes
└── <subdir>/AGENTS.md  ← per-subdir rules (most-specific wins) — optional

~/.warp/                ← Warp local app state (themes, launch configs) — platform-owned
~/.claude.json | .mcp.json | .codex/config.toml ← MCP servers auto-discovered by Warp — shared
```

**Rules precedence** (most specific first): current subdir rules file → repo-root rules file →
Global Rules. Put gald3r enforcement in the **root `AGENTS.md`/`WARP.md`** so it applies
project-wide and auto-loads each session (no manual pin required).

**Format note**: Warp rules are single-file markdown — there is **no** `.mdc`/glob-scoped rules
folder. gald3r's `g-rl-*` set must be flattened into the root rules file.

## 3. gald3r Integration

### Project Rules (primary surface)

The main integration is the root rules file — gald3r enforcement flattened into `AGENTS.md`
(and/or `WARP.md`). Warp auto-applies it within the project every session, so gald3r rules
*do* inject (this corrects the prior pitfall claim). Because Warp has no `.mdc`/glob rule folder,
the fine-grained `g-rl-*` set is consolidated into the root rules file.

### MCP (shared config)

Warp auto-discovers existing MCP servers from `~/.claude.json`, root `.mcp.json`, and
`.codex/config.toml` — any MCP server gald3r already relies on is reused with no Warp-specific
re-config.

### Shell Profile (optional context helper)

Add to `~/.bashrc` / `~/.zshrc` / PowerShell profile for ambient gald3r context:

```bash
# gald3r context for Warp sessions
export GALD3R_ACTIVE_TASK=$(cat .gald3r/TASKS.md 2>/dev/null | grep '\[🔄\]' | head -1)
export GALD3R_PROJECT=$(cat .gald3r/PROJECT.md 2>/dev/null | head -3)
```

### Install

```bash
node bin/install.js --only warp
```

Writes the root rules file (`AGENTS.md`/`WARP.md`) and Warp-compatible setup instructions.

### Warp Drive Workflows (manual / cloud-only)

Warp Drive Workflows are cloud-backed and are **not** installed from on-disk `commands/g-*.md`.
Create gald3r operational Workflows manually in Warp Drive if desired:
- `gald3r status` — show active tasks
- `gald3r commit T{id}` — commit with task reference
- `gald3r new-task` — launch task creation

## 5. Common Pitfalls

- Use **ALL-CAPS** filenames (`AGENTS.md` / `WARP.md`) — Warp only recognizes the rules file in all caps. If both exist, `WARP.md` wins.
- Warp rules are single-file markdown — do not expect `.mdc`/glob-scoped rule files to be honored; flatten `g-rl-*` into the root rules file.
- Warp Drive Workflows/Notebooks are cloud-backed and are NOT installed from the repo — recreate them manually if you want gald3r operations as Workflows.
- No lifecycle hooks fire on Warp (session-start/inbox/pre-commit `.ps1` hooks don't run); custom slash commands and agent hooks are an open feature request (warpdotdev/warp #6857). Run gald3r hook scripts manually in the terminal.

## 6. Known Gaps (terminal-only constraints vs. IDE capabilities)

Warp is an AI terminal, not an IDE — several Cursor-reference primitives have no on-disk home.
Honest status (✅ verified · ⚠️ partial · ❌ absent · ❓ untested):

| Capability | Status | Gap detail |
|---|---|---|
| Rules | ⚠️ | Real auto-applying `AGENTS.md`/`WARP.md` project rules, BUT single-file markdown — no `.mdc`/glob-scoped rule folder. `g-rl-*` must be flattened. |
| MCP | ✅ | First-class; auto-discovers `~/.claude.json` / `.mcp.json` / `.codex/config.toml`. (Active server set per-machine, untested in CI.) |
| Commands | ⚠️ | Warp Drive Workflows exist (parameterized sequences) but are cloud-only and not installed from `commands/g-*.md`. Custom slash commands = open RFC #6857. |
| Skills | ❌ | No on-disk `SKILL.md` discovery/auto-load. Warp Drive is the only analogue and is cloud-managed, not repo-installable. |
| Agents | ❌ | No agent-definition-file discovery. Active AI / Agent Mode / Cloud Agents are *modes* of one assistant, not loadable gald3r agent files. |
| Hooks | ❌ | No lifecycle-event/hook system; agent hooks are an open RFC (#6857). Cloud Agents trigger agent runs, not local `.ps1` hooks. |

**Hard gaps (not achievable from the repo today): on-disk agents, on-disk skills, extensible
on-disk commands, lifecycle hooks.** Clean fits: **project Rules** and **MCP**. See
`PLATFORM_SPEC.md` for the full 9-section assessment and verification evidence.
